COMPOSITION  — "these components form / are used to produce X"
               Typical members: ACTIVE_SPECIES + CONDITION[processing] → ELECTRODE (product)
               or multiple ACTIVE_SPECIES → mixture
               Typical cardinality: 3–8

OPERATION    — "under this material+condition setup, these metrics were measured"
               Typical members: ELECTRODE + MEMBRANE + ACTIVE_SPECIES
               + CONDITION[operating] + METRIC[performance]
               Highest cardinality: 5–15

DEGRADATION  — "under these conditions, this failure or mechanism occurred"
               Typical members: DEGRADATION entity + affected materials
               + triggering CONDITIONS + affected METRICS
               Typical cardinality: 3–7

COMPARISON   — "under a shared criterion, these candidates show different values"
               Typical members: 2+ candidates of SAME ontology type
               + criterion METRIC + each candidate's measured value
               Typical cardinality: 4–10

RULE 5 — COMPARISON trigger (CRITICAL)
═══════════════════════════════════════════════════════════════════════

When the text presents a SERIES of same-type entities measured
under a shared criterion, you MUST generate a COMPARISON hyperedge.
Watch for these patterns:

  "A: x; B: y; C: z"
  "RCC: 0.614 m²/g; TCC: 0.706 m²/g; BCC: 0.800 m²/g;
  TBCC: 0.904 m²/g"
  markdown tables where rows share the same column headers

For each such series:
  (a) Include ALL candidates and ALL their metric values in the vertices list.
  (b) State the comparison criterion and conclusion in the description.
